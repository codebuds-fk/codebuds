Async request tracking using Node's async_hooks

Kashyap

async_hooks provides a way to attach event handlers for all async work that Node does: think HTTP request handling, gzip compression, any Promise execution. Using this we can implement a standalone context store, similar to React's context flow, on the server side, with which we can store data like request Id for logging. The beauty of this API is that we don't have to pass the context object around to each function. In this talk, I'll go through how we used the API to do just that.

Pre-render route and inline critical css to improve LCP scores

Sivakumar

In FPG, the current architecture is that the page gets rendered after downloading the assets.
< div id="app" >
<script src="vendor"/>
<script src="app"/>
Instead, we can prerender the routes with some tools like puppeteer and inline the critical css.
<style>...critical </style>
<div id="app">...app html...<div>
<link rel="preload" href="uncritical.css"/>
<script src="vendor"/>
<script src="app"/>
With this approach, the html content gets rendered more quickly delivering better user experience to the consumers. This approach is supported by an npm package named "react-snap".
